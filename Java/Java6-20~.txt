6-26 static 메서드와 인스턴스 메서드

사진파일 1

메서드 앞에 static이 붙어있으면 static 메서드
메서드 앞에 static이 붙어있지 않으면 인스턴스 메서드



static 메서드와 인스턴스 메서드 차이점

인스턴스 메서드
- 인스턴스 생성 후, '참조변수.메서드이름()'으로 호출
- 인스턴스 멤버(iv, im)와 관련된 작업을 하는 메서드, (매개변수 x)
- 메서드 내에서 인스턴스 변수(iv) 사용가능



static 메서드(클래스메서드)
- 객체생성없이 '클래스이름.메서드이름()'으로 호출
- 인스턴스 멤버(iv, im)와 관련없는 작업을 하는 메서드, (매개변수 o)
- 메서드 내에서 인스턴스 변수(iv) 사용불가

6-32 생성자(constructor)
- 인스턴스가 생성될 때마다 호출되는 '인스턴스 초기화 메서드'
- 인스턴스 생성시 수행할 작업(iv 초기화)에 사용

생성자의 조건
1. 생성자의 이름은 클래스의 이름과 같아야함
2. 생성자는 리턴 값이 없음(void를 안붙임)
3. 모든 클래스는 반드시 하나 이상의 생성자를 가져야 함
